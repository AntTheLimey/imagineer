## Project Context

Imagineer is a TTRPG campaign management platform that:
- Handles campaign data including GM-only "GM notes"
- Manages entity relationships and canon conflicts
- Imports content from external sources (Evernote, Google Docs)
- Stores game system schemas and character data
- Must maintain strict separation between GM and player data

**High-Risk Areas:**
- `/cmd/server/` - API server handling auth and data access
- `/internal/api/` - API handlers processing user input
- `/internal/importers/` - External content processing
- `/client/src/` - Web UI handling user input and sessions

**Critical Security Requirement:**
GM notes (GM-only content) must NEVER be exposed to players. This requires
careful access control at every layer.

## Security Best Practices Reference

### Secure Coding Patterns

**Input Validation**
```go
// Always validate and sanitize input
func ValidateEntityName(name string) error {
    if len(name) < 1 || len(name) > 200 {
        return errors.New("invalid name length")
    }
    if !regexp.MustCompile(`^[a-zA-Z0-9\s\-']+$`).MatchString(name) {
        return errors.New("invalid name characters")
    }
    return nil
}
```

**Parameterized Queries**
```go
// SECURE: Use parameterized queries
row := db.QueryRow(ctx, "SELECT * FROM entities WHERE id = $1", entityID)

// VULNERABLE: String concatenation
row := db.QueryRow(ctx, "SELECT * FROM entities WHERE id = " + entityID) // NEVER DO THIS
```

**Credential Handling**
```go
// Hash passwords with bcrypt
hashedPassword, err := bcrypt.GenerateFromPassword([]byte(password), bcrypt.DefaultCost)

// Generate secure tokens
token := make([]byte, 32)
_, err := crypto_rand.Read(token)
```

**Error Handling**
```go
// Don't expose internal errors to users
if err != nil {
    log.Error("database error", "error", err, "entityID", entityID)
    return errors.New("an error occurred") // Generic message to user
}
```
